# Probe run — visual-aids tech-stack claims (2026-07-15)

Kept output for the `## Tech Stack` section's `verified-by-probe` tags. Two load-bearing rendering
claims, each probed against the real thing rather than read about. 30-minute box; both claims closed
inside it.

Constraint (i) held: no probe added a dependency to any product manifest or lockfile, and nothing
executable ships. The spike tree lived in scratch and is gone; only this record and the shape
artifact beside it survive.

## Probe 1 — GitHub's Markdown REST API does NOT render mermaid to SVG (refines the claim)

**Claim under test:** "GitHub renders a fenced mermaid block as a diagram."

**Method:** posted a mermaid fence to the real API, `POST /markdown` via `gh api`.

**Output (verbatim, trimmed):**

```html
<div class="highlight highlight-source-mermaid"><pre><span class="pl-k">flowchart</span> ...
```

**Finding:** the markdown pipeline **syntax-highlights** the fence and tags it
`highlight-source-mermaid` — it emits **no SVG**. Rendering is not done by the markdown renderer.
Had the probe stopped here and been read charitably, it would have "confirmed" the claim while
actually showing the API does not render diagrams at all. This is why probe 2 exists.

## Probe 2 — a real served GitHub page DOES carry a mermaid render container (confirms)

**Method:** fetched a real public blob page known to contain mermaid fences
(`github.com/mermaid-js/mermaid/blob/develop/README.md`, 468207 bytes served) and grepped the
served HTML for render hooks.

**Output:**

```
     10 data-type="mermaid"
```

Context around the first hit (unescaped):

```html
... class="...render-needs-enrichment position-relative"
    data-host="https://viewscreen.githubusercontent.com"
    data-src="https://viewscreen.githubusercontent.com/markdown/mermaid?docs_host=https%3A%2F%2Fdocs.github.com"
    data-type="mermaid" aria-label="mermaid rendered output container">
  <div class="js-render-enrichment-target" data-json="{"data":"flowchart LR ...
```

**Finding:** GitHub serves one **render-enrichment container per fence** (10 fences -> 10
containers), carrying the diagram source in `data-json` and pointing at
`viewscreen.githubusercontent.com/markdown/mermaid` for the rendered output. The
`aria-label="mermaid rendered output container"` is GitHub's own wording.

**Verdict: CONFIRMED** — a `mermaid` fence in a committed markdown file renders as a diagram on
GitHub, via client-side enrichment rather than the markdown pipeline. Consistent with GitHub's
official docs (checked 2026-07-15), which state rendering works in Markdown files, Issues,
Discussions, pull requests, and wikis.

**Carried consequence:** the renderer is the **host's**, not ours. We pin no mermaid version and
cannot: GitHub runs its own. Hence the tech-stack constraint to author only long-stable core diagram
types, never the newly-added ones.

## Probe 3 — a scratch visual needs no rendering library (confirms)

**Claim under test:** "a scratch visual can compare two shapes side by side in one self-contained
page with no dependency" — load-bearing, because the alternative (loading a diagram library from a
CDN) would break both self-containment and NC-5.

**Method:** authored a minimal side-by-side comparison page using inline CSS and inline SVG only,
then scanned it for any external reference and parsed it for well-formedness.

**Output:**

```
--- external-reference scan (must be zero: any http(s)://, src=, or <link> = a network dep):
        (no matches)
--- structural parse (well-formed? element counts):
   parsed OK, tags: {'style': 1, 'div': 3, 'svg': 2, 'rect': 5, 'text': 5, 'line': 2}
--- bytes: 1805 (single file, opens via file:// URL)
```

**Verdict: CONFIRMED** — 1805 bytes, zero external references (`https?://`, `src=`, `<link>`,
`<script>`, `@import`, `url(...)` all absent), well-formed. Side-by-side comparison of two candidate
shapes needs no library, no CDN, no server, and no build step.

The probe page is kept beside this record as `scratch-visual-shape-2026-07-15.html` — it doubles as
the shape reference for what a scratch visual is (inline SVG, inline CSS, a dark-mode media query,
`role="img"` + `aria-label` per panel).
